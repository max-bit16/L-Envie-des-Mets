# .impeccable — Design Constitution
# L'Envie des Mets

## Voice
Sobre, précis, chaleureux. Zéro fioriture, zéro slop marketing.
Chaque mot justifié par le contexte du restaurant.

## Visual Laws

### 1. Whitespace is the first ingredient
- Sections breathe: `128px` vertical padding minimum on desktop
- Never compress spacing to fit more content — cut content instead
- Margins between heading → body → CTA: never less than `24px`

### 2. Type hierarchy is sacred
- One H1 per page, always Fraunces 400
- Eyebrow → H2 → body paragraph — never skip levels
- Italic `<em>` in headings for accent, never bold serif
- Prices: Fraunces, `--wood` color, never sans-serif

### 3. Color discipline
- Forest (`#2A3A2E`) = trust, action, brand — use sparingly
- Wood (`#A87B5D`) = warmth, detail, accent — underlines, stars, rules
- Stone (`#6B6258`) = supporting text only — never headlines
- Never introduce a new color without updating `index.css` tokens

### 4. Photography over illustration
- Every section that can take a photo, should
- Overlay gradients must preserve legibility: `rgba(26,26,26,0.45)` minimum on text
- Never crop faces at awkward boundaries — `object-position: center 30%` for portraits

### 5. Motion is subtle
- `.reveal`: 600ms, `cubic-bezier(0.4, 0, 0.2, 1)` — never bouncier
- Hover transitions: 150–240ms only
- Hero parallax: max 40px travel — never carnival scroll effects
- Always gate on `prefers-reduced-motion`

### 6. Interaction is honest
- Every button does one thing, named by its action
- No fake urgency ("Dernières places!" without data)
- Phone number must be real and clickable on mobile (`href="tel:..."`)

## Audit Checklist (run before push)

### Typography
- [ ] No orphaned words on headings at common viewport widths
- [ ] Eyebrow present on every major section
- [ ] All `<em>` in headings are intentional italics (not error)

### Spacing
- [ ] Hero padding-bottom ≥ 64px mobile
- [ ] No section padding below 72px on mobile

### Color
- [ ] No hardcoded hex values outside `index.css` variables
- [ ] All overlay text passes WCAG AA (≥4.5:1 contrast)

### Assets
- [ ] All `<img>` have meaningful `alt` text
- [ ] No placeholder images (vite.svg, react.svg) in production build
- [ ] Favicon is custom (not Vite default)

### Accessibility
- [ ] Skip link present and functional
- [ ] All interactive elements have `focus-visible` ring
- [ ] Sticky bar phone link uses `href="tel:..."`

### Performance
- [ ] `npm run build` exits 0 errors, 0 warnings
- [ ] No unused imports in TSX files
- [ ] Hero image < 200KB (compressed)

## Anti-patterns (instant reject)

- Box shadows on content cards
- `border-radius` > 4px on buttons or containers
- Any font-weight ≥ 600 on serif headings
- Gradient backgrounds on non-photo sections
- Inline `style={}` for anything the CSS system already handles
- Toast notifications or popups for a restaurant landing page
- Animations that trigger on hover on mobile
